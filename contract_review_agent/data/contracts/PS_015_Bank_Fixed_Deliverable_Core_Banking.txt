PROJECT OVERVIEW

Migration from legacy core banking system to cloud-native platform supporting retail and SME banking operations for 2.5 million customers and 2,500 staff across 75 branches.

DELIVERABLES

Phase 1 - Assessment and Planning (12 weeks)
- Current state architecture and data assessment
- Migration strategy and approach (big bang vs phased)
- Data migration analysis (accounts, transactions, customer data)
- Integration mapping (channels, payments, lending, treasury)
- Regulatory impact assessment (PRA, FCA)
- Business continuity and rollback planning
- Detailed project plan and resourcing

Phase 2 - Design and Build (20 weeks)
- Target state architecture design
- Core banking platform configuration (Thought Machine Vault)
- Customer data migration tooling and process
- Channel integration (online, mobile, branch, ATM)
- Payment system integration (Faster Payments, BACS, CHAPS)
- Lending platform integration
- Regulatory reporting framework
- Security and fraud prevention systems
- Data reconciliation framework

Phase 3 - Testing and Assurance (16 weeks)
- Functional testing (all products and services)
- Integration testing (end-to-end scenarios)
- Performance and load testing (3x peak capacity)
- Disaster recovery and failover testing
- Security and penetration testing
- Regulatory compliance testing
- User acceptance testing (branch staff and customers)
- Parallel running (4 weeks)
- Independent audit and assurance

Phase 4 - Migration and Go-Live (8 weeks)
- Pre-migration readiness assessment
- Customer communication campaign
- Staff training (2,500 users)
- Data migration execution (3-day weekend)
- Go-live and cutover management
- Hypercare support (4 weeks, 24/7)
- Post-implementation review
- Regulatory reporting to PRA/FCA
